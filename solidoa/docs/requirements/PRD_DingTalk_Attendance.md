# SolidOA 钉钉考勤集成需求文档 (PRD)

> 版本：V1.0
> 日期：2026-05-26
> 状态：草稿

---

## 一、需求背景

### 1.1 现有考勤系统

当前系统考勤功能：
- ✅ 员工打卡（上班/下班）
- ✅ 打卡位置校验
- ✅ 补卡申请审批
- ✅ 月度考勤统计
- ❌ **缺少：钉钉考勤数据同步、加班数据对接、考勤异常处理**

### 1.2 钉钉考勤优势

```
┌─────────────────────────────────────────────────────────────────┐
│                    钉钉考勤生态                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  员工使用钉钉App打卡 → 数据自动同步到钉钉云                      │
│         ↓                                                        │
│  企业管理员在钉钉后台配置考勤规则                                │
│         ↓                                                        │
│  系统通过钉钉API获取打卡记录、加班数据                           │
│         ↓                                                        │
│  SolidOA进行考勤数据展示、统计、分析                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**钉钉考勤核心能力：**
- **实时打卡**：员工在钉钉App内打卡，数据实时上传
- **位置打卡**：支持GPS定位，防止代打卡
- **加班记录**：加班申请自动同步到钉钉
- **考勤统计**：自动生成考勤报表

---

## 二、需求概述

### 2.1 核心目标

将SolidOA考勤系统与钉钉考勤深度集成，实现：
1. **数据同步**：实时获取钉钉打卡记录、加班数据
2. **统一入口**：员工使用钉钉打卡，系统自动同步
3. **异常处理**：考勤异常自动检测，触发补卡流程
4. **数据展示**：考勤数据可视化，辅助管理决策

### 2.2 用户故事

| 角色 | 用户故事 | 优先级 |
|------|----------|--------|
| 员工 | 我在钉钉打卡后，希望在SolidOA看到打卡记录 | P0 |
| 员工 | 我在钉钉提交加班申请后，希望同步到SolidOA | P0 |
| 员工 | 我有考勤异常时，希望系统提醒我提交补卡 | P1 |
| 管理员 | 我希望看到所有员工的钉钉打卡统计数据 | P1 |
| 管理员 | 我希望系统自动标记迟到、早退、缺卡情况 | P1 |
| 管理员 | 我希望导出月度考勤报表（包含钉钉数据） | P2 |

## 三、功能需求

### 3.1 钉钉考勤数据同步

#### 3.1.1 打卡记录同步

**功能描述：**
- 定时从钉钉拉取员工打卡记录
- 支持按部门、员工、日期范围筛选
- 自动过滤无效打卡数据

**数据来源：**
```java
// 钉钉打卡记录API
GET /cgi/attendance/getrecord
{
  "userId": "员工钉钉ID",
  "workDateFrom": "2026-05-01",
  "workDateTo": "2026-05-26"
}
```

**同步策略：**
| 策略 | 说明 | 优先级 |
|------|------|--------|
| 实时同步 | 钉钉Webhook推送+定时轮询 | P0 |
| 增量同步 | 仅同步新增/变化的数据 | P1 |
| 全量同步 | 每日凌晨全量拉取一次 | P2 |

#### 3.1.2 加班记录同步

**功能描述：**
- 从钉钉获取员工加班申请记录
- 同步加班时长、加班类型
- 支持加班转调休/加班费计算

**数据来源：**
```java
// 钉钉加班记录API
GET /cgi/attendance/get_overtime_list
{
  "userIdList": ["员工钉钉ID列表"],
  "startTime": 1709280000000,  // 毫秒时间戳
  "endTime": 1709366400000
}
```

### 3.2 考勤异常检测

#### 3.2.1 异常类型

| 异常类型 | 检测规则 | 处理方式 |
|----------|----------|----------|
| 迟到 | 上班打卡时间 > 规定时间 | 记录迟到，触发提醒 |
| 早退 | 下班打卡时间 < 规定时间 | 记录早退，触发提醒 |
| 缺卡 | 当日无打卡记录 | 记录缺卡，触发补卡申请 |
| 旷工 | 连续3天无打卡记录 | 记录旷工，触发警告 |
| 请假重叠 | 打卡时间与请假时间重叠 | 标记异常，管理员审核 |

#### 3.2.2 异常通知

**通知触发条件：**
- 每日18:00检测当日考勤异常
- 每周一上午发送上周考勤汇总
- 每月1日发送上月考勤报告

**通知方式：**
- 钉钉消息推送
- SolidOA站内消息
- 邮件通知（可选）

## 四、技术方案

### 4.1 钉钉考勤API

#### 4.1.1 打卡记录获取

```java
/**
 * 获取员工打卡记录
 * API: TOP API - taobao.attendance.getrecord
 */
public DingTalkAttendanceVo getAttendanceRecords(
    String userId,
    LocalDate startDate,
    LocalDate endDate
) {
    // 1. 调用钉钉考勤API
    DingTalkResult result = dingTalkClient.execute(
        "taobao.attendance.getrecord",
        Map.of(
            "userid", userId,
            "start_date", startDate.toString(),
            "end_date", endDate.toString()
        )
    );

    // 2. 解析返回数据
    List<AttendanceRecord> records = parseRecords(result);

    // 3. 存储到本地数据库
    return convertToVo(records);
}
```

#### 4.1.2 加班记录获取

```java
/**
 * 获取员工加班记录
 * API: TOP API - taobao.attendance.getovertimelist
 */
public List<OvertimeRecord> getOvertimeRecords(
    List<String> userIds,
    LocalDateTime startTime,
    LocalDateTime endTime
) {
    // 调用钉钉加班API
    DingTalkResult result = dingTalkClient.execute(
        "taobao.attendance.getovertimelist",
        Map.of(
            "userid_list", String.join(",", userIds),
            "start_time", startTime.toInstant().toEpochMilli(),
            "end_time", endTime.toInstant().toEpochMilli()
        )
    );

    return parseOvertimeRecords(result);
}
```

### 4.3 定时同步任务

| 任务 | 周期 | 说明 |
|------|------|------|
| 打卡记录同步 | 每5分钟 | 增量同步当日打卡数据 |
| 加班记录同步 | 每30分钟 | 增量同步加班数据 |
| 考勤异常检测 | 每日18:00 | 检测当日异常并通知 |
| 月度报表生成 | 每月1日 00:00 | 生成上月考勤报表 |
| 全量数据同步 | 每日02:00 | 全量同步上月数据 |

## 五、非功能需求

### 5.1 性能要求

| 指标 | 要求 |
|------|------|
| 数据同步延迟 | ≤ 5分钟 |
| 页面加载时间 | ≤ 2秒 |
| 同时在线用户 | ≥ 100人 |
| API响应时间 | ≤ 500ms |

### 5.2 安全要求

| 要求 | 说明 |
|------|------|
| 数据脱敏 | 员工敏感信息（位置、打卡详情）需脱敏 |
| 权限控制 | 仅管理员可查看他人考勤数据 |
| 日志审计 | 所有考勤数据访问需记录日志 |

### 5.3 兼容性

| 项目 | 要求 |
|------|------|
| 钉钉版本 | 钉钉6.0及以上 |
| 手机系统 | iOS 12+、Android 9+ |
| 网络要求 | 4G/WiFi环境下使用 |

**前端：**

- `solidoa-web/src/views/dashboard/` - 考勤数据看板

**数据库：**
- `oa_attendance` - 增加钉钉关联字段
- 新增 `oa_attendance_sync_log` 表

### 6.2 需新增的表

```sql
-- 考勤同步日志表
CREATE TABLE oa_attendance_sync_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    sync_type VARCHAR(20) NOT NULL COMMENT '同步类型:CLOCK/OVERTIME',
    sync_start_time DATETIME NOT NULL COMMENT '同步开始时间',
    sync_end_time DATETIME COMMENT '同步结束时间',
    status VARCHAR(20) DEFAULT 'RUNNING' COMMENT '状态:SUCCESS/FAILED/RUNNING',
    records_count INT DEFAULT 0 COMMENT '同步记录数',
    error_message TEXT COMMENT '错误信息',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_sync_type (sync_type),
    INDEX idx_status (status)
);

-- 钉钉打卡记录表（扩展）
ALTER TABLE oa_attendance_record
ADD COLUMN dingtalk_record_id VARCHAR(64) COMMENT '钉钉打卡记录ID',
ADD COLUMN dingtalk_device_id VARCHAR(64) COMMENT '钉钉设备ID',
ADD COLUMN location_result VARCHAR(20) COMMENT '定位结果:LOCATED/OUT_OF_RANGE/NORMAL',
ADD COLUMN base_check_time DATETIME COMMENT '基准打卡时间';
```
