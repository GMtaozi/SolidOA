## 1. 考勤管理

### 1.1 功能概览

| 功能   | 说明            | 优先级 |
| ---- | ------------- | --- |
| 补卡申请 | 异常打卡补录        | P0  |
| 请假申请 | 年假、事假、病假、婚假等  | P0  |
| 加班申请 | 工作日/休息日/节假日加班 | P0  |
| 出差申请 | 出差行程报备        | P0  |
| 外勤申请 | 外出办公登记        | P1  |
| 考勤统计 | 月度考勤汇总        | P0  |

### 1.2 补卡申请

**表单字段：**

| 字段   | 类型  | 必填  | 说明                 |
| ---- | --- | --- | ------------------ |
| 补卡日期 | 日期  | 是   | 需要补卡的日期            |
| 补卡类型 | 单选  | 是   | 上班打卡/下班打卡          |
| 补卡时间 | 时间  | 是   | 实际到岗/离岗时间          |
| 补卡原因 | 下拉  | 是   | 忘记打卡/外勤未打卡/系统故障/其他 |
| 详细说明 | 文本  | 是   | 具体原因描述             |
| 证明材料 | 附件  | 否   | 外勤照片、截图等           |

**审批流程：**

```
申请人 → 直属主管 → [人事确认] → 完成
```

**业务规则：**

- 每月补卡次数限制：5次（可配置）
- 当月补卡需在次月5日前提交
- 超过次数需总经理审批

### 1.3 请假申请

**表单字段：**

| 字段   | 类型   | 必填  | 说明                    |
| ---- | ---- | --- | --------------------- |
| 假期类型 | 下拉   | 是   | 年假/事假/病假/婚假/产假/陪产假/丧假 |
| 开始时间 | 日期时间 | 是   | 请假开始                  |
| 结束时间 | 日期时间 | 是   | 请假结束                  |
| 请假时长 | 数字   | 自动  | 系统计算（小时）              |
| 请假原因 | 文本   | 是   | 具体原因                  |
| 证明材料 | 附件   | 否   | 病假需提供医院证明             |

**假期余额：**

| 假期类型 | 年度额度  | 是否带薪 | 说明     |
| ---- | ----- | ---- | ------ |
| 年假   | 5-15天 | 是    | 按工龄计算  |
| 事假   | 无限制   | 否    | 扣除当日工资 |
| 病假   | 无限制   | 部分   | 需医院证明  |
| 婚假   | 3天    | 是    | 晚婚加7天  |
| 产假   | 158天  | 是    | 难产加15天 |
| 陪产假  | 15天   | 是    | -      |

**审批流程：**

```
≤3天：申请人 → 直属主管 → 完成
＞3天：申请人 → 直属主管 → 部门负责人 → 完成
＞5天：申请人 → 直属主管 → 部门负责人 → 人事确认 → 完成
```

### 1.4 加班申请

**表单字段：**

| 字段   | 类型   | 必填  | 说明                |
| ---- | ---- | --- | ----------------- |
| 加班类型 | 单选   | 是   | 工作日加班/休息日加班/节假日加班 |
| 开始时间 | 日期时间 | 是   | 加班开始              |
| 结束时间 | 日期时间 | 是   | 加班结束              |
| 加班时长 | 数字   | 自动  | 系统计算（小时）          |
| 加班原因 | 文本   | 是   | 具体工作内容            |
| 补偿方式 | 单选   | 是   | 调休/加班费            |

**加班补偿标准：**

| 加班类型  | 补偿标准          |
| ----- | ------------- |
| 工作日加班 | 1.5倍工资 或 等时调休 |
| 休息日加班 | 2倍工资 或 等时调休   |
| 节假日加班 | 3倍工资（不可调休）    |

**审批流程：**

```
申请人 → 直属主管 → 完成
```

### 1.5 考勤统计

**统计维度：**

| 统计项   | 说明      |
| ----- | ------- |
| 应出勤天数 | 当月工作日天数 |
| 实际出勤  | 实际打卡天数  |
| 迟到次数  | 迟到打卡次数  |
| 早退次数  | 早退打卡次数  |
| 请假时长  | 各类假期累计  |
| 加班时长  | 加班累计    |
| 补卡次数  | 补卡累计    |

## 2. 差旅报销

| 功能   | 说明     | 优先级 |
| ---- | ------ | --- |
| 出差申请 | 出差行程报备 | P0  |
| 差旅报销 | 出差费用报销 | P0  |

### 2.2 出差申请

**表单字段：**

| 字段   | 类型  | 必填  | 说明             |
| ---- | --- | --- | -------------- |
| 出差事由 | 文本  | 是   | 出差目的           |
| 出发地  | 文本  | 是   | 出发城市           |
| 目的地  | 文本  | 是   | 到达城市           |
| 开始日期 | 日期  | 是   | 出差开始           |
| 结束日期 | 日期  | 是   | 出差结束           |
| 出差天数 | 数字  | 自动  | 系统计算           |
| 出差人数 | 数字  | 是   | 同行人数           |
| 交通方式 | 多选  | 是   | 飞机/高铁/火车/自驾/大巴 |
| 预计费用 | 金额  | 否   | 预估总费用          |
| 备注   | 文本  | 否   | 其他说明           |

**审批流程：**

```
≤3天：申请人 → 直属主管 → 完成
＞3天：申请人 → 直属主管 → 部门负责人 → 完成
跨省出差：申请人 → 直属主管 → 部门负责人 → 总经理 → 完成
```

### 2.3 差旅报销

**表单字段：**

| 字段    | 类型  | 必填  | 说明         |
| ----- | --- | --- | ---------- |
| 关联出差单 | 选择  | 是   | 关联已审批的出差申请 |
| 报销明细  | 子表  | 是   | 费用明细列表     |
| 报销总额  | 金额  | 自动  | 系统汇总       |

**报销明细子表：**

| 字段   | 类型  | 必填  | 说明                      |
| ---- | --- | --- | ----------------------- |
| 费用类型 | 下拉  | 是   | 交通费/住宿费/餐饮费/市内交通/通讯费/其他 |
| 发生日期 | 日期  | 是   | 费用发生日期                  |
| 金额   | 金额  | 是   | 实际金额                    |
| 发票号码 | 文本  | 否   | 发票号码                    |
| 发票图片 | 附件  | 是   | 发票照片                    |
| 备注   | 文本  | 否   | 费用说明                    |

**差旅标准（可配置）：**

| 职级   | 住宿标准/天 | 餐饮标准/天 | 交通标准      |
| ---- | ------ | ------ | --------- |
| 普通员工 | 300元   | 100元   | 高铁二等座/经济舱 |
| 主管级  | 400元   | 120元   | 高铁二等座/经济舱 |
| 经理级  | 500元   | 150元   | 高铁一等座/经济舱 |
| 总监级  | 600元   | 200元   | 高铁一等座/商务舱 |
| 总经理  | 800元   | 300元   | 不限        |

**审批流程：**

```
申请人 → 直属主管 → 财务审核 → 完成
```

**业务规则：**

- 超标准费用需说明原因
- 发票必须真实有效
- 报销需在出差结束后15天内提交

### 2.4 费用标准配置

| 配置项  | 说明              |
| ---- | --------------- |
| 城市等级 | 一线/二线/三线城市，不同标准 |
| 职级标准 | 不同职级的报销上限       |
| 费用类型 | 各类费用的默认标准       |
| 审批规则 | 超标审批流程          |

| 财务专用章 | 财务部   | 财务总监  | 财务相关   |
| 人事专用章 | 人力资源部 | HR总监  | 人事文件   |

**表单字段：**

| 字段   | 类型  | 必填  | 说明       |
| ---- | --- | --- | -------- |

| 用印事由 | 文本  | 是   | 使用原因     |

| 期望日期 | 日期  | 是   | 需要使用日期   |
| 备注   | 文本  | 否   | 其他说明     |

**审批流程：**

| 印章类型  | 审批流程                                |
| ----- | ----------------------------------- |
| 部门章   | 申请人 → 部门负责人 → 行政部 → 完成              |
| 人事专用章 | 申请人 → 部门负责人 → HR总监 → 行政部 → 完成       |
| 财务专用章 | 申请人 → 部门负责人 → 财务总监 → 行政部 → 完成       |
| 合同专用章 | 申请人 → 部门负责人 → 法务审核 → 总经理 → 行政部 → 完成 |
| 公章    | 申请人 → 部门负责人 → 总经理 → 行政部 → 完成        |

**业务规则：**

- 用印需提前1个工作日申请
- 盖章时需核对文件与申请附件一致
- 用印后行政部需登记用印记录
- 原则上不外借印章

## 6. 公共流程设计

### 6.1 审批节点类型

| 节点类型 | 说明         | 支持操作        |
| ---- | ---------- | ----------- |
| 审批节点 | 需要审批人同意/拒绝 | 同意、拒绝、转办、加签 |
| 抄送节点 | 通知相关人员     | 查看（自动通过）    |
| 条件节点 | 根据条件分支     | 自动判断        |
| 结束节点 | 流程结束       | -           |

### 6.2 审批人规则

| 规则     | 说明         |
| ------ | ---------- |
| 指定人员   | 固定审批人      |
| 指定角色   | 某角色的人员     |
| 直属主管   | 申请人的直属上级   |
| 部门负责人  | 申请人所在部门负责人 |
| 连续多级主管 | 逐级审批       |
| 发起人自选  | 申请人选择审批人   |

### 6.3 审批操作

| 操作  | 说明          |
| --- | ----------- |
| 同意  | 通过审批，流转下一节点 |
| 拒绝  | 驳回申请，流程结束   |
| 转办  | 转给其他人审批     |
| 加签  | 添加额外审批人     |
| 退回  | 退回到某个节点     |
| 催办  | 催促审批人处理     |
| 撤回  | 申请人撤回申请     |

### 6.4 超时处理

| 策略   | 说明      |
| ---- | ------- |
| 自动通过 | 超时自动通过  |
| 自动拒绝 | 超时自动拒绝  |
| 催办提醒 | 发送催办通知  |
| 自动转办 | 转给指定人处理 |

## 7. 数据库设计建议

### 7.1 核心表结构

```sql
-- 审批流程实例表
CREATE TABLE wf_process_instance (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    process_key VARCHAR(64) COMMENT '流程标识',
    business_id BIGINT COMMENT '业务单据ID',
    business_type VARCHAR(32) COMMENT '业务类型',
    title VARCHAR(255) COMMENT '流程标题',
    initiator_id BIGINT COMMENT '发起人ID',
    status VARCHAR(20) COMMENT 'running/completed/cancelled',
    create_time DATETIME,
    update_time DATETIME
);

-- 审批任务表
CREATE TABLE wf_task (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    instance_id BIGINT COMMENT '流程实例ID',
    node_name VARCHAR(64) COMMENT '节点名称',
    assignee_id BIGINT COMMENT '处理人ID',
    status VARCHAR(20) COMMENT 'pending/approved/rejected',
    opinion VARCHAR(500) COMMENT '审批意见',
    complete_time DATETIME,
    create_time DATETIME
);

-- 考勤打卡记录表
CREATE TABLE att_clock_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT COMMENT '员工ID',
    clock_date DATE COMMENT '打卡日期',
    clock_type VARCHAR(10) COMMENT '上班/下班',
    clock_time DATETIME COMMENT '打卡时间',
    location VARCHAR(255) COMMENT '打卡地点',
    latitude DECIMAL(10,6),
    longitude DECIMAL(10,6),
    status VARCHAR(20) COMMENT '正常/迟到/早退',
    create_time DATETIME
);

-- 假期余额表
CREATE TABLE att_leave_balance (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT COMMENT '员工ID',
    year INT COMMENT '年度',
    leave_type VARCHAR(20) COMMENT '假期类型',
```
