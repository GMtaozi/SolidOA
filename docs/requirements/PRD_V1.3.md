## 1. 产品概述

### 1.1 产品定位

SolidOA 是一款面向中小型企业的轻量级办公自动化系统，专注于审批流程、协作办公和财务管理三大核心模块。

### 1.2 现有功能

- ✅ 审批流程：请假、报销、用印、采购
- ✅ 协作办公：消息中心、通讯录、日程
- ✅ 财务管理：报销审核、预算管理
- ✅ 考勤管理：打卡、统计、钉钉同步
- ✅ 钉钉集成：免登、消息推送

### 1.3 新增需求

本次迭代新增以下功能模块：

| 序号 | 功能模块 | 优先级 | 说明 |
|------|----------|--------|------|
| 1 | 考勤规则完善 | P1 | 加班、补卡、出差、假期等规则 |
| 2 | 工资管理 | P2 | 工资审批、工资条制作与发放 |
| 3 | 审批记录 | P2 | 所有审批类型的统一记录查询 |
| 4 | 系统管理菜单 | P1 | 用户、部门、角色、规则、权限等 |

## 2. 考勤规则需求

### 2.1 功能背景

参考钉钉考勤功能，完善SolidOA考勤模块，支持多种考勤场景。

### 2.2 加班规则

#### 2.2.1 加班类型

| 加班类型 | 类型码 | 说明 | 加班工资 |
|----------|--------|------|----------|
| 工作日加班 | WORKDAY | 工作日延长工作时间 | 按审批单时长计算 |
| 休息日加班 | REST_DAY | 周六周日加班 | 按审批单时长计算 |
| 节假日加班 | HOLIDAY | 国家法定节假日加班 | 按审批单时长计算 |

#### 2.2.2 加班申请规则

| 规则项 | 说明 |
|--------|------|
| 审批要求 | 所有加班必须审批通过后生效 |
| 时长计算 | 加班时长以审批单填写时间为准 |
| 最小单位 | 0.5小时 |
| 紧急加班 | 可事后补申请，需说明原因 |

#### 2.2.3 加班调休规则

| 规则项 | 说明 | 管理员可配置 |
|--------|------|--------------|
| 补偿方式 | 加班可选择兑换为调休时间或支付加班工资 | ✅ |
| 调休比例 | 统一1:1（加班1小时 = 调休1小时） | ✅ 可配置 |
| 调休有效期 | 加班后6个月内使用 | ✅ 可配置 |

> **注意**：当前调休比例统一为1:1，管理员可在后台修改规则。

### 2.3 补卡规则

#### 2.3.1 补卡类型

| 补卡类型 | 类型码 | 说明 |
|----------|--------|------|
| 缺卡 | MISSING | 上下班均未打卡 |
| 迟到 | LATE | 上班迟到未打卡 |
| 早退 | EARLY_LEAVE | 下班早退未打卡 |

#### 2.3.2 补卡规则

| 规则项 | 说明 | 管理员可配置 |
|--------|------|--------------|
| 补卡次数 | 每月可提交5次 | ✅ 可配置 |
| 计时周期 | 每月的1日起算 | ❌ 固定 |
| 补卡时限 | 可申请最近31个工作日内的补卡 | ✅ 可配置 |
| 审批要求 | 需直属上级审批 | ✅ 可配置 |
| 证明材料 | 需说明补卡原因（可选附件） | ✅ 可配置 |

> **注意**：补卡规则均可由管理员在后台配置修改。

#### 2.3.3 补卡场景

| 补卡场景 | 说明 | 是否需要审批 |
|----------|------|--------------|
| 上班忘打卡 | 上班时间漏打卡 | 是 |
| 下班忘打卡 | 下班时间漏打卡 | 是 |
| 外出忘打卡 | 因公外出无法打卡 | 是 |
| 出差忘打卡 | 出差期间漏打卡 | 是 |

### 2.4 出差规则

#### 2.4.1 出差申请

- 出差需提前提交申请
- 出差目的地、事由、预计时长必填
- 可关联差旅报销单

#### 2.4.2 出差考勤

- 出差期间自动标记为出差状态
- 出差期间无需打卡
- 出差返回后自动恢复打卡

#### 2.4.3 出差类型

| 出差类型 | 说明 | 差旅补贴 |
|----------|------|----------|
| 本地出差 | 当天往返 | 50元/天 |
| 短途出差 | 1-3天 | 150元/天 |
| 长途出差 | 3天以上 | 200元/天 |

### 2.5 假期规则

#### 2.5.1 假期类型

| 假期类型 | 类型码 | 时效 | 说明 |
|----------|--------|------|------|
| 年假 | ANNUAL | 按工龄计算 | 入职满1年享受 |
| 病假 | SICK | 无上限 | 需提供医院证明 |
| 事假 | PERSONAL | 每年10天 | 扣工资 |
| 婚假 | MARRIAGE | 法定天数 | 领证后一年内 |
| 产假 | MATERNITY | 法定天数 | 158天起 |
| 陪产假 | PATERNITY | 法定天数 | 15-30天 |
| 丧假 | FUNERAL | 法定天数 | 1-5天 |
| 工伤假 | WORK_INJURY | 依伤情 | 需工伤认定 |

#### 2.5.2 年假计算规则

| 工龄 | 年假天数 |
|------|----------|
| 1年内 | 0天 |
| 1-5年 | 5天 |
| 5-10年 | 10天 |
| 10-20年 | 15天 |
| 20年以上 | 20天 |

#### 2.5.3 假期余额管理

- 假期余额按年度计算
- 年底未使用年假可结转最多5天
- 请假优先使用年假余额
- 销假后返还对应假期

### 2.6 考勤异常处理

#### 2.6.1 异常类型

| 异常类型 | 说明 | 处理方式 |
|----------|------|----------|
| 迟到 | 上班打卡晚于规定时间 | 扣款/记录 |
| 早退 | 下班打卡早于规定时间 | 扣款/记录 |
| 旷工 | 无故缺勤 | 扣双倍工资 |
| 漏打卡 | 上下班任一未打卡 | 补卡申请 |

#### 2.6.2 迟到处理规则

| 迟到时长 | 处理方式 |
|----------|----------|
| 1-30分钟 | 扣款50元 |
| 31-60分钟 | 扣款100元 |
| 61-120分钟 | 扣半天工资 |
| 120分钟以上 | 扣一天工资 |

### 2.7 考勤配置

#### 2.7.1 班次设置

| 班次 | 上班时间 | 下班时间 | 适用部门 |
|------|----------|----------|----------|
| 标准班 | 09:00 | 18:00 | 默认 |
| 弹性班 | 09:00-10:00 | 18:00-19:00 | 弹性部门 |
| 值班班 | 08:00 | 17:00 | 前台等 |

#### 2.7.2 工作日设置

- 默认：周一至周五
- 可配置：周六上班、周日休息
- 法定节假日自动排除

## 3. 工资管理需求

### 3.1 功能背景

实现工资审批与工资条制作发放功能，参考钉钉工资条模块。

### 3.2 工资审批

#### 3.2.1 工资表制作

- 每月由财务人员制作工资表
- 工资项目包含：基本工资、岗位工资、绩效、补贴、扣款等
- 支持Excel导入/导出
- 支持工资模板设置

#### 3.2.2 工资审批流程

```
财务制作 → HR审核 → 主管审批 → 发放
```

- 审批节点可配置
- 支持驳回修改
- 审批通过后锁定，不可修改

#### 3.2.3 工资项目定义

| 项目类别 | 工资项 | 说明 |
|----------|--------|------|
| 应发工资 | 基本工资 | 合同约定 |
| | 岗位工资 | 职级对应 |
| | 绩效工资 | 月度绩效 |
| | 加班费 | 加班工资 |
| | 补贴 | 餐补/交通补 |
| 扣款 | 社保公积金 | 个人部分 |
| | 个税 | 代扣代缴 |
| | 迟到扣款 | 考勤扣款 |
| | 其他扣款 | 借款等 |
| 实发工资 | 应发工资 - 扣款合计 | 到账金额 |

### 3.3 工资条发放

#### 3.3.1 发放方式

| 发放方式 | 说明 | 优势 |
|----------|------|------|
| 工资条卡片 | 发送到员工APP | 实时查看 |
| 邮件发送 | 发送到员工邮箱 | 留存记录 |
| 纸质工资条 | 打印发放 | 传统需求 |

#### 3.3.2 工资条内容

- 员工基本信息：姓名、部门、工号
- 计薪周期：XXXX年XX月
- 各项工资明细：项目、金额
- 汇总信息：应发、扣款、实发
- 备注说明

#### 3.3.3 发放记录

- 记录发放时间、方式、状态
- 员工查收确认
- 发送失败重试机制

### 3.4 员工查询

#### 3.4.1 历史查询

- 查询最近12个月工资条
- 支持按月份筛选
- 支持导出PDF

#### 3.4.2 工资统计

- 个人工资明细汇总
- 年度收入证明
- 个税申报数据

### 3.5 数据安全

- 工资数据仅员工本人可查看
- 管理员分权限查看
- 查看记录审计日志
- 敏感数据加密存储

## 5. 审批记录功能

### 5.1 功能背景

提供统一的审批记录查询功能，方便用户查看历史审批数据。

### 5.2 审批记录类型

| 记录类型 | 说明 | 包含审批 |
|----------|------|----------|
| 差旅报销 | TRAVEL | 出差相关报销 |
| 请假 | LEAVE | 所有假期类型 |
| 采购 | PURCHASE | 采购申请 |
| 工资 | SALARY | 工资审批 |
| 用章 | STAMP | 用印申请 |
| 加班 | OVERTIME | 加班申请 |
| 出差 | BUSINESS_TRIP | 出差申请 |
| 补卡 | REPAIR_CARD | 补卡申请 |
| 外出 | OUTING | 外出申请 |

### 5.3 查询维度

#### 5.3.1 按用户查询

- 我发起的：查看我提交的所有申请
- 我审批的：查看我审批过的所有申请
- 待我审批：查看需要我审批的申请

#### 5.3.2 按时间查询

- 今日
- 本周
- 本月
- 自定义日期范围

#### 5.3.3 按状态查询

- 全部
- 审批中
- 已通过
- 已拒绝
- 已撤回

### 5.4 记录详情

- 申请基本信息：申请人、时间、类型
- 审批流程图：节点进度可视化
- 审批详情：每个节点的审批人、意见、时间
- 抄送记录：抄送人列表、阅读状态
- 操作日志：各操作时间点

### 5.5 导出功能

- 支持导出Excel
- 导出字段可配置
- 批量导出

## 附录

### A. 用户故事

#### 用户故事1：员工申请加班

**作为** 员工
**我希望** 能够提交加班申请
**以便** 记录加班时间并获得调休或加班费

**验收标准：**
- 能够选择加班类型、时间和时长
- 能够选择调休或加班费
- 提交后进入审批流程
- 审批通过后系统记录加班

#### 用户故事2：员工查看审批记录

**作为** 员工
**我希望** 能够查看我的所有审批记录
**以便** 了解申请进度和处理结果

**验收标准：**
- 能够查看我发起的所有申请
- 能够按类型、状态、时间筛选
- 能够查看每个申请的审批进度
- 能够导出审批记录

#### 用户故事3：HR制作工资条

**作为** HR
**我希望** 能够制作并发放工资条
**以便** 员工及时了解薪资明细

**验收标准：**
- 能够制作工资表
- 能够审批工资
- 能够发放工资条
- 员工能够查询工资条

| 安全 | 工资数据加密存储、分权限查看 |
| 可用性 | 支持批量导入导出 |
| 可扩展 | 工资项可配置化 |

### C. 优先级定义

| 优先级 | 说明 |
|--------|------|
| P0 | 核心功能，必须实现 |
| P1 | 重要功能，影响业务 |
| P2 | 增强功能，提升体验 |
| P3 | 优化功能，可延后 |

### D. 自动计算机制

所有业务规则均支持管理员配置，系统自动计算执行：

| 功能 | 自动计算 | 管理员可配置 |
|------|----------|--------------|
| 年假天数 | ✅ 按工龄自动计算 | ✅ 规则可配置 |
| 补卡次数 | ✅ Redis计数器自动校验 | ✅ 次数上限可配置 |
| 加班调休 | ✅ 按加班类型自动计算 | ✅ 兑换比例可配置 |
| 工资个税 | ✅ 累计预扣法自动计算 | ❌ 国税规定固定 |
| 考勤异常 | ✅ 打卡与班次自动对比 | ✅ 宽限时间可配置 |
| 历史归档 | ✅ 定时任务自动归档 | ✅ 归档周期可配置 |

**设计原则：**
- 管理员修改规则 → 系统自动重新计算
- 无需人工干预
- 支持规则模拟验证
