# SolidOA 回归测试方案

> 版本：V1.0
> 日期：2026-05-24
> 依据：SSD V1.1 第9.6节「测试策略」

---

## 1. 测试范围与目标

### 1.1 测试范围

覆盖以下核心业务路径：
1. 用户认证（登录、JWT刷新、登出、钉钉免登）
2. 系统管理（用户CRUD、部门树、角色分配、权限校验）
3. 请假审批（完整流程、加签、转交、驳回、撤回、催办）
4. 费用报销（完整流程、预算检查）
5. 考勤打卡（打卡、迟到、补卡）
6. 消息通知（WebSocket、钉钉工作通知）
7. 文件服务（上传、下载、预览）
8. 钉钉同步（用户、部门、考勤）

### 1.2 测试通过标准

| 测试类型 | 覆盖率目标 | 通过标准 |
|----------|------------|----------|
| 单元测试 | ≥ 70% | 核心业务逻辑通过 |
| 集成测试 | 关键路径 100% | 所有用例通过 |
| 回归测试 | 全场景覆盖 | 关键路径通过率 ≥ 95% |
