# SolidOA 错误码规范

## 错误码结构

```json
{
  "code": 400,
  "message": "参数错误",
  "data": null,
  "timestamp": 1699999999999
}
```

## 错误码分类

| 错误码范围 | 类别 | 说明 |
|-----------|------|------|
| 200-299 | 成功 | 请求成功 |
| 400-499 | 客户端错误 | 参数错误、未认证、无权限 |
| 500-599 | 服务端错误 | 系统异常、数据库错误 |

---

## 客户端错误 (4xx)

### 400 - 参数错误

| 子错误码 | 说明 | 常见场景 |
|---------|------|----------|
| 40001 | 参数为空 | 必填参数未传 |
| 40002 | 参数格式错误 | 手机号格式、邮箱格式 |
| 40003 | 参数超出范围 | 分页参数 < 0 |
| 40004 | 参数类型错误 | 期望数字收到字符串 |

### 401 - 未认证

| 子错误码 | 说明 | 解决方案 |
|---------|------|----------|
| 40101 | Token 过期 | 刷新 Token 或重新登录 |
| 40102 | Token 无效 | 检查 Token 格式 |
| 40103 | Token 为空 | 检查请求头是否携带 Token |

### 403 - 无权限

| 子错误码 | 说明 | 解决方案 |
|---------|------|----------|
| 40301 | 无访问权限 | 检查用户角色 |
| 40302 | 无操作权限 | 需要更高权限 |
| 40303 | 跨部门访问 | 只能访问本部门数据 |
| 40304 | 无文件访问权限 | 仅上传者或关联用户可访问 |

### 404 - 资源不存在

| 子错误码 | 说明 | 常见场景 |
|---------|------|----------|
| 40401 | 用户不存在 | 检查 userId |
| 40402 | 部门不存在 | 检查 deptId |
| 40403 | 请假单不存在 | 检查 leaveId |
| 40404 | 文件不存在 | 检查 fileId |

### 409 - 冲突

| 子错误码 | 说明 | 常见场景 |
|---------|------|----------|
| 40901 | 重复提交 | 已有相同的申请 |
| 40902 | 状态冲突 | 当前状态不允许此操作 |
| 40903 | 数据已变更 | 乐观锁冲突，需要刷新 |

### 429 - 请求过于频繁

| 子错误码 | 说明 | 解决方案 |
|---------|------|----------|
| 42901 | 登录限流 | 登录失败次数过多 |
| 42902 | 接口限流 | 请求频率超出限制 |

---

## 服务端错误 (5xx)

### 500 - 系统异常

| 子错误码 | 说明 | 处理建议 |
|---------|------|----------|
| 50001 | 数据库错误 | 检查数据库连接 |
| 50002 | Redis 错误 | 检查 Redis 连接 |
| 50003 | MQ 发送失败 | 检查 RabbitMQ 连接 |
| 50004 | 文件存储错误 | 检查 MinIO 连接 |

### 502 - 网关错误

| 子错误码 | 说明 | 处理建议 |
|---------|------|----------|
| 50201 | 服务不可用 | 检查下游服务状态 |
| 50202 | 服务超时 | 下游服务响应超时 |

### 503 - 服务维护

| 子错误码 | 说明 | 处理建议 |
|---------|------|----------|
| 50301 | 服务升级中 | 稍后重试 |
| 50302 | 服务停机维护 | 关注公告 |

## 业务错误 (1xxx)

### 考勤模块 (1000-1999)

| 错误码 | 说明 |
|--------|------|
| 1001 | 打卡地点超出范围 |
| 1002 | 设备指纹验证失败 |
| 1003 | 补卡申请重复 |
| 1004 | 考勤统计数据异常 |

### 工作流模块 (2000-2999)

| 错误码 | 说明 |
|--------|------|
| 2001 | 审批状态不允许操作 |
| 2002 | 审批人不是当前处理人 |
| 2003 | 流程实例不存在 |
| 2004 | 消息发送失败 |

### 财务模块 (3000-3999)

| 错误码 | 说明 |
|--------|------|
| 3001 | 预算不足 |
| 3002 | 预算已被占用 |
| 3003 | 报销金额超出范围 |
| 3004 | 付款确认失败 |

### 文件模块 (4000-4999)

| 错误码 | 说明 |
|--------|------|
| 4001 | 文件类型不支持 |
| 4002 | 文件 Magic Number 验证失败 |
| 4003 | 文件大小超出限制 |
| 4004 | 无权限访问文件 |

### 钉钉集成 (5000-5999)

| 错误码 | 说明 |
|--------|------|
| 5001 | 钉钉 API 调用失败 |

| 5003 | 钉钉签名验证失败 |
| 5004 | 钉钉配置未设置 |

## 前端错误处理示例

```javascript
import { ElMessage } from 'element-plus'

// 统一错误处理
export function handleError(error) {
  const code = error.code || error.response?.data?.code

  switch (true) {
    case code >= 400 && code < 500:
      // 客户端错误
      ElMessage.error(error.response?.data?.message || '请求失败')
      break
    case code >= 500:
      // 服务端错误
      ElMessage.error('服务器异常，请稍后重试')
      break
    case code === 429:
      // 限流
      ElMessage.warning('请求过于频繁，请稍后重试')
      break
    default:
      ElMessage.error('未知错误')
  }
}
```

## 日志追踪

当出现错误时，请提供 `X-Trace-Id` 以便定位问题：

```
请求头: X-Trace-Id: abc123def456
错误响应中会包含 traceId 字段
```
