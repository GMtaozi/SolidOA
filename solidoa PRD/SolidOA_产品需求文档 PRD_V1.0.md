## 1. 产品概述

### 1.1 产品简介

| 项目 | 内容 |
|------|------|
| **产品名称** | SolidOA |
| **产品类型** | 企业内部办公自动化系统 |
| **目标用户** | 企业内部员工（约20人规模） |
| **核心功能** | 审批流、协作办公、财务管理 |
| **部署方式** | 本地私有化部署 |
| **移动端支持** | iOS/Android APP + 微信小程序 |

### 1.2 产品定位

SolidOA 是一款面向中小型企业的轻量级办公自动化系统，专注于审批流程、协作办公和财务管理三大核心模块。系统采用微服务架构，支持本地私有化部署，确保数据安全可控。

### 5.4 数据库设计

#### 核心表结构

```sql
-- 用户表
CREATE TABLE sys_user (
    id VARCHAR(36) PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    real_name VARCHAR(50),
    mobile VARCHAR(20),
    email VARCHAR(100),
    dept_id VARCHAR(36),
    status TINYINT DEFAULT 1,
    dingtalk_userid VARCHAR(100),
    create_time DATETIME,
    update_time DATETIME
);

-- 部门表
CREATE TABLE sys_department (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id VARCHAR(36),
    leader_id VARCHAR(36),
    sort INT DEFAULT 0,
    dingtalk_id VARCHAR(100),
    create_time DATETIME
);

-- 角色表
CREATE TABLE sys_role (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    code VARCHAR(50) NOT NULL UNIQUE,
    description VARCHAR(200),
    create_time DATETIME
);

-- 用户角色表
CREATE TABLE sys_user_role (
    user_id VARCHAR(36),
    role_id VARCHAR(36),
    PRIMARY KEY (user_id, role_id)
);

-- 权限表
CREATE TABLE sys_permission (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    code VARCHAR(100) NOT NULL UNIQUE,
    type VARCHAR(20),
    url VARCHAR(200),
    parent_id VARCHAR(36)
);

-- 角色权限表
CREATE TABLE sys_role_permission (
    role_id VARCHAR(36),
    permission_id VARCHAR(36),
    PRIMARY KEY (role_id, permission_id)
);

-- 请假申请表
CREATE TABLE oa_leave (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) NOT NULL,
    leave_type VARCHAR(20) NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    days DECIMAL(5,1) NOT NULL,
    reason TEXT,
    status VARCHAR(20) DEFAULT 'PENDING',
    process_instance_id VARCHAR(100),
    create_time DATETIME,
    update_time DATETIME
);

-- 报销申请表
CREATE TABLE oa_expense (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) NOT NULL,
    expense_type VARCHAR(50) NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    reason TEXT,
    attachments VARCHAR(500),
    status VARCHAR(20) DEFAULT 'PENDING',
    process_instance_id VARCHAR(100),
    create_time DATETIME,
    update_time DATETIME
);

-- 考勤记录表
CREATE TABLE oa_attendance (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) NOT NULL,
    check_time DATETIME NOT NULL,
    check_type VARCHAR(20),
    location VARCHAR(200),
    source VARCHAR(20),
    dingtalk_id VARCHAR(100),
    create_time DATETIME
);

-- 消息表
CREATE TABLE oa_message (
    id VARCHAR(36) PRIMARY KEY,
    title VARCHAR(100),
```

### 8.4 启动脚本

```bash
#!/bin/bash
# start-all.sh

echo "===== SolidOA 启动 ====="

# 启动MySQL
systemctl start mysqld

# 启动Redis
systemctl start redis

# 启动Nacos
cd /opt/solidoa/nacos && sh bin/startup.sh -m standalone &

# 等待Nacos启动
sleep 15

# 启动微服务
for jar in solidoa-gateway solidoa-system solidoa-workflow solidoa-collaboration solidoa-finance solidoa-file solidoa-dingtalk; do
    nohup java -jar /opt/solidoa/${jar}.jar > /opt/solidoa/logs/${jar}.log 2>&1 &
    echo "Started: ${jar}"
done

echo "===== SolidOA 启动完成 ====="
echo "Gateway: http://localhost:8080"
echo "Nacos:   http://localhost:8848"
```
